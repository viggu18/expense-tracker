# Qoder IDE Rules

## Task Execution
- Do not generate long explanations after each task
- Write a small explanation after each task (maximum 3 sentences)
- Focus on implementing the requested features without verbose documentation

## File Generation
- Do not write test files unless specifically requested
- Do not proactively create documentation files (*.md) or README files
- Only create documentation files if explicitly requested by the User
- NEVER create files unless they're absolutely necessary for achieving the goal

## Code Formatting
- Ignore formatting issues during implementation
- Focus on functionality over formatting unless specifically requested

## File Management
- Ask before deleting any file
- ALWAYS prefer editing an existing file to creating a new one
- NEVER create files unless they're absolutely necessary for achieving your goal

## Communication Style
- Be concise and direct
- Provide only essential information
- Avoid unnecessary elaboration
